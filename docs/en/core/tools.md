# Tools

Eight built-in tools, plus the `agent` tool when subagents are enabled.

## Naming

`verb_object`, lowercase. Names the model already understands — no implementation details, no namespaces.

## Tool set

| Tool | Required args | Optional args | Risk |
|------|--------------|---------------|------|
| `find_files` | — | `path`, `pattern`, `max_depth` | `read` |
| `read_file` | `path` | `offset`, `limit` | `read` |
| `search_text` | `pattern` | `path`, `include`, `view` | `read` |
| `edit_file` | `path`, `old_text`, `new_text`, `brief` | `replace_all` | `edit` |
| `write_file` | `path`, `content`, `brief` | — | `edit` |
| `run_command` | `command`, `timeout`, `brief` | — | `command` |
| `remember_memory` | `scope`, `kind`, `text` | — | `edit` |
| `forget_memory` | `scope`, `text` | — | `edit` |
| `agent` | `name`, `prompt` | — | `read` |

### `find_files`

Browse the file tree. `pattern` is a file glob (`*.md`, `docs/**/*.md`). `max_depth` limits recursion. Returns relative paths.

### `read_file`

Read a file with `line<TAB>text` line numbers. `offset`/`limit` for pagination. Returns a short stub if the file hasn't changed since the last `read_file` in this session.

### `search_text`

Regex search over file content. `view`: `files` (file list), `lines` (matching lines), `count` (per-file counts). Default `files`.

### `edit_file`

Precise text replacement. `old_text` must match uniquely. `replace_all: true` for bulk replacements. `brief` describes the edit in 3-5 words. Requires a prior `read snapshot`.

### `write_file`

Write complete file content — create or overwrite. Overwrites require a prior full-file `read snapshot`. `brief` describes the file in 3-5 words.

### `run_command`

Execute a local command. `timeout` in seconds, required. Default cwd is workspace. `brief` describes the command in 3-5 words. stdout/stderr are merged.

### `remember_memory` / `forget_memory`

Append or remove a bullet in `memory.md`. `scope`: `global` or `project`. `kind`: `how_to_work`, `what_is_true`, or `where_to_look`.

### `agent`

Dispatch a task to a named subagent in an isolated child session. Available agents are listed in the catalog injected into `runtime_context`.

## Declaration

Every tool declares:

| Field | Description |
|-------|-------------|
| `name` | Stable public name |
| `description` | Short description for the model |
| `input_schema` | JSON Schema, root must be `object` |
| `risk` | `read` / `edit` / `command` |
| `read_only` | Whether the tool never modifies state |
| `max_result_chars` | Hard cap on `model_content` length |

## Result envelope

All tools return:

| Field | Description |
|-------|-------------|
| `status` | `ok` / `error` / `blocked` / `cancelled` |
| `summary` | English one-liner describing the outcome |
| `model_content` | Content for the model to continue reasoning |
| `truncated` | Whether `model_content` was cut at `max_result_chars` |
| `structured` | Optional machine-readable detail |

`summary` is derived from execution facts, not generated by the model. `model_content` is the evidence for the next turn.
