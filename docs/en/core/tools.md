# Tools

Eleven built-in tools, plus the `agent` tool when subagents are enabled and `use_skill` when skills are enabled.

## Naming

`verb_object`, lowercase. Names the model already understands — no implementation details, no namespaces.

## Tool set

| Tool | Required args | Optional args | Risk |
|------|--------------|---------------|------|
| `find_files` | — | `path`, `pattern`, `max_depth` | `read` |
| `read_file` | `path` | `offset`, `limit` | `read` |
| `search_text` | `pattern` | `path`, `include`, `view`, `offset`, `limit`, `context` | `read` |
| `fetch_url` | `url` | — | `read` |
| `fetch_web` | `url`, `prompt`, `model_tier` | — | `read` |
| `query_session` | — | `search`, `type`, `from_turn`, `to_turn`, `limit` | `read` |
| `edit_file` | `path`, `old_text`, `new_text`, `brief` | `replace_all` | `edit` |
| `write_file` | `path`, `content`, `brief` | — | `edit` |
| `run_command` | `command`, `timeout`, `brief` | — | `command` |
| `remember_memory` | `scope`, `kind`, `text` | — | `edit` |
| `forget_memory` | `scope`, `text` | — | `edit` |
| `agent` | `name`, `prompt` | — | `read` |

### `find_files`

Browse the file tree. `pattern` is a file glob (`*.md`, `docs/**/*.md`). `max_depth` limits recursion. Returns relative paths.

### `read_file`

Read a file with `line<TAB>text` line numbers. `offset`/`limit` for pagination. Returns a short stub if the file hasn't changed since the last `read_file` in this session.

### `search_text`

Regex search over file content. `view`: `files` (file list), `lines` (matching lines), `count` (per-file counts). Default `files`. `offset`/`limit` for paginating large result sets (default: 0/100). `context` shows N lines before and after each match (only with `view=lines`, default: 0).

### `edit_file`

Precise text replacement. `old_text` must match uniquely. `replace_all: true` for bulk replacements. `brief` describes the edit in 3-5 words. Requires a prior `read snapshot`.

### `write_file`

Write complete file content — create or overwrite. Overwrites require a prior full-file `read snapshot`. `brief` describes the file in 3-5 words.

### `run_command`

Execute a local command. `timeout` in seconds, required. Default cwd is workspace. `brief` describes the command in 3-5 words. stdout/stderr are merged.

### `fetch_url`

Download a file from a URL to a temp directory (`/tmp/kuku-fetch/`). Returns metadata: status code, content-type, size, and file path. Enforces a 50MB size limit and rejects URLs with embedded credentials or non-HTTP(S) schemes. Read-only — does not modify workspace files.

### `fetch_web`

Fetch a web page, extract readable content via readability, convert to Markdown, and optionally summarize with a secondary LLM call. Content under 50KB is returned directly; larger pages are summarized using the specified `model_tier`. Results are cached (200 entries, 5-minute TTL). HTML body limited to 10MB.

### `query_session`

Query historical session events that have scrolled out of the current context window. Only useful after a context handoff. Filters: `type` (UserInput, ModelResponse, ToolCall, ToolResult, Handoff, HandoffTrigger), `search` (full-text keyword), `from_turn`/`to_turn` (relative turn range, 0 = most recent), `limit` (default 20). Individual event content is truncated to 500 chars; total output capped at 8000 chars. Read-only.

### `remember_memory` / `forget_memory`

Append or remove a bullet in `memory.md`. `scope`: `global` or `project`. `kind`: `how_to_work`, `what_is_true`, or `where_to_look`.

### `agent`

Dispatch a task to a named subagent in an isolated child session. Available agents are listed in the catalog injected into `runtime_context`.

## Declaration

Every tool declares:

| Field | Description |
|-------|-------------|
| `name` | Stable public name |
| `description` | Short description for the model |
| `input_schema` | JSON Schema, root must be `object` |
| `risk` | `read` / `edit` / `command` |
| `read_only` | Whether the tool never modifies state |
| `max_result_chars` | Hard cap on `model_content` length |

## Result envelope

All tools return:

| Field | Description |
|-------|-------------|
| `status` | `ok` / `error` / `blocked` / `cancelled` |
| `summary` | English one-liner describing the outcome |
| `model_content` | Content for the model to continue reasoning |
| `truncated` | Whether `model_content` was cut at `max_result_chars` |
| `structured` | Optional machine-readable detail |

`summary` is derived from execution facts, not generated by the model. `model_content` is the evidence for the next turn.

## ExecSlot

Every tool call runs in its own `ExecSlot` — a tokio task with an independent cancel token. All slots run concurrently (up to 32 active at once; additional calls are queued). Each slot holds:

| Field | Description |
|-------|-------------|
| `tool_call_id` | Links to the originating `tool.call` event |
| `kind` | `ToolKind` — determines streaming behavior |
| `label` | Human-readable description for host display |
| `cancel` | `Arc<tokio::sync::Notify>` — single-slot cancellation |
| `child_permissions` | Channel map for subagent permission routing |

Slots report events via a shared `mpsc::Sender<(tool_call_id, SlotEvent)>` channel. The `Run` polls this channel in `next()` and converts `SlotEvent` into `UiEvent` for the host.

## ToolKind

Classifies the execution mode of a tool:

| Variant | Description |
|---------|-------------|
| `Simple` | Builtin tools (find_files, read_file, etc.). No intermediate output. |
| `Agent { child_session_id }` | Subagent child session. Streams real-time `ToolEvent` via `ToolOutput`. |
| `Command { pid }` | `run_command` execution. Streams stdout/stderr via `ToolEvent::Stdout`/`Stderr`. |

## ToolEvent

Events produced inside an `ExecSlot`:

| Variant | Description |
|---------|-------------|
| `TextDelta` | Streaming text from child model (agent slots) |
| `ThinkingDelta` | Streaming thinking from child model |
| `ToolStart` | Child session tool use started |
| `ToolOutput` | Nested `ToolEvent` from child session |
| `ToolEnd` | Child session tool use completed |
| `Stdout` | Command stdout line |
| `Stderr` | Command stderr line |
| `PermissionRequested` | Child session permission routed to parent |
| `Error` | Child session error |

`ToolEvent` is converted to `UiEvent` by `map_ui_to_tool_event` before reaching the host.

## Single-tool cancellation

`Run::cancel_tool(tool_call_id)` cancels one running tool by notifying its slot's cancel token. The slot's spawned task exits via `tokio::select!` biased cancellation. Other slots continue unaffected. Returns `true` if a matching slot was found.
